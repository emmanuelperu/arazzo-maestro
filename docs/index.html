arazzo-maestro

Lint, render & test Arazzo workflows. One Go binary. Eco-designed, accessible by default.

arazzo-maestro turns Arazzo workflow specs into three things the rest of your team can actually use: validation that fails your CI when something is wrong, standalone HTML pages anyone can open in a browser, and runnable end-to-end tests you point at any environment, no server, no build, no JavaScript.

Three subcommands

One score, three batons

The maestro inspects the score, conducts it, and rehearses it: lint validates, view renders, test generates runnable tests. No other moving parts.

lint

Validate, then fail CI on demand

Three passes: the official OAI JSON Schema, internal semantic rules (unique IDs, $steps.x.outputs.y references), and a cross-file check that every step operationId actually exists in the referenced OpenAPI contract. Exit 0/1 plus parseable findings, offline, deterministic.

view

Render to standalone HTML

One self-contained page per workflow: START → steps → END, runtime expressions highlighted, success criteria, request bodies. ~3 kB gzipped, opens offline in any browser. Commit it, or ship it to GitHub Pages.

test

Generate runnable tests from a workflow

test gen e2e turns a workflow into Hurl files: each step becomes a request, outputs become captures, success criteria become asserts, the host stays a {{baseUrl}} variable. test run e2e runs them against any environment with an optional HTML report. Performance tests (k6) are coming next.

Built for after authoring

Editor plugins help you write Arazzo. We handle everything after.

Validation in CI, an artifact you can commit and deploy, zero runtime dependency. Complementary to your IDE plugin, the same person can run both.

01

Three-pass linter

Official JSON Schema, semantic rules, and cross-file operationId resolution against your OpenAPI contracts.

02

Standalone HTML

No server, no client-side rendering, no JavaScript. One file, ~18 kB raw / ~3 kB gzipped, viewable offline.

03

Test generation

Turn a workflow into runnable end-to-end tests. test gen/run e2e emits Hurl files ({{baseUrl}}, captures, asserts) and runs them with an HTML report. Performance tests (k6) are coming next.

04

Eco-designed by rule

System fonts only, inline SVG, minimal requests, a single static binary on a FROM scratch image. Sobriety is enforced, not aspirational.

05

Accessible by default

WCAG 2.2 AA contrasts, semantic HTML, visible focus, no info by colour alone, prefers-reduced-motion honoured.

06

Themeable

Light and dark built in; drop a themes.yml to set a default or add your own. Custom themes are contrast-audited at load time.

07

One binary, no runtime

No Python, no Node, no cgo. Trivial cross-compilation, instant startup, a tiny Docker image. Install in one command.

Get going

Quick start

Go 1.23+ for a source install, or run it straight from Docker, nothing else to set up.

# Install (Go 1.23+)
$ go install github.com/emmanuelperu/arazzo-maestro/cmd/arazzo-maestro@latest

# Validate
$ arazzo-maestro lint workflow.yaml

# Render (light + dark)
$ arazzo-maestro view workflow.yaml -o dist/light
$ arazzo-maestro view workflow.yaml -o dist/dark --theme dark
$ open dist/light/index.html

# Generate + run e2e tests (Hurl)
$ arazzo-maestro test run e2e workflow.yaml --base-url https://staging.example.com
# Or via Docker, FROM scratch image
$ docker run --rm -v "$PWD":/work \
    ghcr.io/emmanuelperu/arazzo-maestro:latest \
    view /work/workflow.yaml -o /work/dist

# List available themes
$ arazzo-maestro view --list-themes
dark
light (default)
