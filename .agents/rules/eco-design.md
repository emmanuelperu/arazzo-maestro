# Eco-design rules

> **Guiding principle**: every kilobyte, every request, every
> watt-hour counts. The HTML generated by `arazzo-maestro` is consumed
> by devs and ops, often on a laptop or mobile on the move. The binary
> runs on a user's machine or in a CI. Be sober by default.

## Rules

### R1, No external resource that isn't strictly necessary

The generated HTML loads **only** the resources required to render. As
of today: only the Tailwind CDN (eventually to be internalised, see
R8).

**Forbidden without an explicit waiver:**

- Google Fonts or any remote font service
- JS libraries (analytics, frameworks, polyfills) on the output side
- Decorative raster images, use inline SVG or unicode glyphs
- Trackers, beacons, third-party scripts
- Self-hosted fonts as long as a system stack works

**Rationale**: every third-party request costs a TCP/TLS handshake,
~30–100 kB transferred, and can track the user (GDPR). A system stack
is instantaneous, free, and already loaded.

### R2, Fonts: system stacks only

`font:` in `themes.yml` is limited to `sans` / `serif` / `mono`. No
Google Fonts, no remote `@font-face`. See Plan.md > Themes feature for
the cross-justification with accessibility.

### R3, Prefer static over dynamic

Output is generated **once** at build time, not on every visit. No
server, no client-side rendering, no dynamic fetches. The HTML file is
viewable offline after generation.

### R4, Inline SVG over raster images

For icons / decorations: inline SVG (cacheable, scalable, zero extra
request). If a raster is strictly necessary, use WebP/AVIF with
explicit `width`/`height`.

### R5, No gratuitous animations

No CSS transition or animation that doesn't serve understanding. An
animation = CPU and battery. Respect `prefers-reduced-motion: reduce`
where transitions are needed (focus, hover…).

### R6, Go binary: small, static, no cgo

- No cgo (breaks cross-compilation and bloats the `FROM scratch`
  Docker image)
- `goreleaser` configured to strip symbols in release
  (`-ldflags="-s -w"`), target size: < 10 MB
- Measured dependencies: prefer stdlib when the gap is small (e.g. no
  ORM framework here, we only parse YAML)

### R7, Measure before claiming sober

Any PR touching HTML rendering must report:

- Generated HTML size for `examples/shop.arazzo.yaml` (gzipped and raw)
- Number of requests at page load (= 1 today, Tailwind CDN)

Regression > 10 % → explicit discussion in the PR.

### R8, Internalise the CSS eventually

The Tailwind CDN is a pragmatic compromise for the POC. Eventually:
extract the strictly used CSS via `tailwindcss --content` at build
time and embed it in the binary (via `//go:embed`), reaching zero
network requests at page load.

### R9, No polling, no unsolicited watcher

If a `watch` mode is added, it must be **opt-in** (`--watch`) and use
`fsnotify` (kernel events), never periodic polling.

## Anti-patterns to refuse

- "Let's add jQuery to handle this small interaction" → write 5 lines
  of vanilla JS, or do without
- "We'll load Inter from Google Fonts for better rendering" → R1 + R2;
  the `ui-sans-serif` stack is already excellent
- "An infinite animation on the logo, it's nice" → R5
- "Preload every OpenAPI operation just in case" → R3, render only
  what's in the Arazzo, nothing more

## Current baseline

- HTML `happy-path-checkout.html`: ~18 kB raw, ~3 kB gzipped
- Requests at page load: 1 (Tailwind CDN)
- Binary: ~6.7 MB (Go 1.23, without `-s -w`)
